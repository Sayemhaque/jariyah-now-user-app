   244→Task: Fix the seek bar not reaching 100% when the video finishes playing.
   245→
   246→Work Log:
   247→- Root cause: in VideoPreview's `onEnded` handler, when the LAST ayat finished playing, the code called `setCurrentTimeMs(0)`. This reset the per-ayat current time to 0, so the seek bar value became `offsets[lastIndex] + 0` = the START of the last ayat, not the end of the whole video. With 3 ayats where the last one starts at ~11s and total is ~16s, the bar showed 0:11/0:16 at completion — matching the user's screenshot exactly.
   248→- Fix: when the last ayat ends, set `currentTimeMs` to the last ayat's `audioDurationMs` (instead of 0). This makes the seek bar value become `offsets[lastIndex] + lastAyatDuration = totalMs` = 100%.
   249→- Also added `ayatList` to the useEffect dependency array so the closure always reads the latest ayat data.
   250→- Verified with Agent Browser: loaded Al-Fatihah ayat 1 (single ayat, ~6 seconds), pressed Play, waited for completion. The seek bar reached 100% and the timecode showed `0:06 / 0:06` — confirmed both via DOM inspection (slider aria-valuenow = 6031, matching total) and via VLM ("seek/progress bar is fully filled to the end (100%)... time shown is 0:06 / 0:06").
   251→- ESLint passes clean (0 errors, 0 warnings)
   252→
