## 2. Workflow to enable the Export button

The "Export video" button is disabled on first load because no ayats have
been fetched. To reach the modal I:

1. Opened the SURAH combobox and selected **1 Al-Fatihah (The Opening,
   7 ayats, Meccan)**.
2. Left FROM=1, TO=3 (default).
3. Clicked "Load ayats" — button label changed to "Reload ayats" and the
   "Export video" button in the header became enabled.
4. Player controls (Previous / Play / Next) also became enabled, confirming
   media was loaded.

No errors during this flow.

### About the "video preview area" question

In the **idle** state (before rendering), the modal does **not** show a
video preview area — only platform/quality pickers and the summary card.
Per `src/components/ExportModal.tsx`, the `VideoPreviewPlayer` component
(rendered with play/pause, seek, volume controls) is mounted only after a
render completes successfully (`RenderStatus === 'done'`). So a preview
area exists, but it only appears post-render. Since the task explicitly
said not to actually render a video, this is the expected behavior and is
not a bug.
